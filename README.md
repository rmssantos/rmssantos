## About Me

I'm a builder who enjoys the whole product loop: shaping the idea, designing the UX, writing the frontend, building the backend, deploying it, tightening security, improving performance, and polishing the small details that make a project feel real.

The common thread in my work is interaction. I like products that respond, teach, simulate, debate, move, or create a small world around the user.

## How I Build

| I care about | What that usually means in practice |
| --- | --- |
| Product usefulness | Start with a workflow, not a tech demo. |
| Interaction quality | Fast feedback, clear states, keyboard/mobile support, and fewer dead ends. |
| Real deployment | CI, hosting, DNS, auth, billing, observability, and boring operational details. |
| Maintainability | Tests where they matter, clear docs, small refactors, and explicit tradeoffs. |
| Security | Safer URLs, CSP, dependency mitigation, auth boundaries, and secret handling. |
